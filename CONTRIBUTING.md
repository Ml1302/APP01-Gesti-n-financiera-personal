# Contribuyendo al Proyecto

¡Gracias por tu interés en contribuir! Estas son las pautas para hacerlo de forma ordenada.

## Reportar Issues

- Usa el [tracker de issues](https://github.com/Ml1302/APP01-Gesti-n-financiera-personal/issues) de GitHub.
- Describe el problema: pasos para reproducir, comportamiento esperado vs real, y entorno.
- Si es una sugerencia, etiquétala como *enhancement*.

## Enviar Pull Requests

1. Haz un fork del repositorio.
2. Crea una rama con nombre descriptivo:
   - `feature/mi-nueva-funcionalidad`
   - `fix/descripcion-del-error`
   - `docs/mejora-documentacion`
3. Mantén los cambios atómicos (un PR por tema).
4. Asegúrate de que el código pase lint y tests (cuando estén configurados).
5. Actualiza la documentación si es necesario.
6. Envía el PR contra la rama `main`.

## Estilo de Código

- Sigue las convenciones del lenguaje/framework existente.
- No añadas comentarios superfluos; el código debe ser autoexplicativo.
- Usa nombres descriptivos para variables, funciones y componentes.

## Commits

- Usa mensajes claros en español o inglés:
  ```
  feat: agregar soporte multimoneda
  fix: corregir cálculo de intereses en préstamos
  docs: actualizar guía de instalación
  ```

## Código de Conducta

Este proyecto sigue el [Código de Conducta](CODE_OF_CONDUCT.md). Al participar, aceptas cumplirlo.
